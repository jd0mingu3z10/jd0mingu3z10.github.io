    1. Introduction to the Web

    Understanding how the web works

  • Internet
    • E-mail, SMS messaging, Application data, World Wide Web
  • Web
    • Runs over the internet
    • Transfers data of the World Wide Web
  • The web is a series of interactions between 2 types of systems, clients and servers.
  • Clients
    • Devices that render web content.
    • Popular clients include browsers, mobile applications, screen readers, and various content aggregators.

      Working with clients and servers

  • When your domain makes a request, it uses the URL (Uniform Resource Locator) to locate the requested content. This request is usually passed on to a domain name server or DNS. The DNS then translates the URL into an IP address. The browser then uses this IP address to locate the host's server and send a request for the content.
  • Dynamics site pages must first be processed by a web application such as PHP, .NET, or Ruby before that content is served to a browser.

    The DNS and why it matters

  • DNS is one of the most important processes on the web and it's also one of the most overlooked.
  • Domain name servers for DNS address one of the fundamental differences between humans and computers.
  • The DNS system is an incredibly complex in fluid database this distributed across servers all over the world.
  • If your site isn't registered properly with a DNS, no one will be able to find you.
  • Multi-name sites need to point to the proper IP.
  • Make sure that you are aware of who site ownership is assigned to.
  • DNS caching can take up to 24 hours.

    Internet protocols

  • Protocols are simple languages that let computers share information with each other.
  • The internet uses a group of protocols commonly referred as to "The Internet Protocol Suite".
  • Transmission control protocol Internet Protocol (TCP/IP)
  • Created in 1973, it has become the main protocol suite used to exchange data over the internet.
  • Internet Protocol Suite
    • Application layer: HTTP, FTP, SMTP, DNS, RIP, SNMP
    • Link layer: Wireless LAN, Ethernet, ATM
    • Transport Layer: TCP, UDP
    • Internet Layer: IPv4, IPv6
  • The internet would not work at all without the listed protocols.

    Exploring HTTP

  • HTTP: hypertext transfer protocol
  • The standard protocol for transferring data over the world wide web.
  • Common web protocols
    • FTP: file transfer protocol, enables transfer of large files
    • SMTP: simple mail transfer protocol, use to send mail
    • RTP: real-Time transfer protocol, standard for audio and video

      Anatomy of a URL

  • URL: Uniform Resource Locator
    • Specialized string of characters that identifies the location of a desired resource.
  • Subdomain
    • A folder that your web content is located in on your web server.

      How browsers work

  • Browsers use HTTP to communicate with web servers and request pages and content.
  • Typical Browser Components
    • Browser UI - Visual Chrome, Address bar, and Other visual elements
  • Internal Engine
    • Directs communication across the various components
  • Rendering Engine
    • Parses HTML and CSS and renders the content on the screen
  • Networking
    • HTTP and other protocol components
  • Data Storage
    • Internal data storage and session management capabilities
  • JavaScipt Interpeter
    • Internal JavaScript engine, executes any client-side scripts

  • In the mid 90's, Netscape Navigator battled with Internet Explorer for market dominance.

3. Back-End Technology

Server-side scripting

  • Any scripting or programming that runs on a web server is referred to as a server-side scripting.
  • Exmples of server-side scripting can be anything from processing of form, managing an online purchase, registering a new user, or even booking a hotel room.
  • PHP, Java, .Net, Cold Fusion and others have given Developers a wide array of tools to choose from when they are creating Dynamic sites. Each language is slightly different.
  • Processing is done on the server, which for high traffic sites increase server load.
  • Processing data usually requires a page refresh on the browser.
  • AJAX and other client side bridges help enhance usability of server side scripting.

    The Importance of PHP

  • In 94 Rasmus Lerdorf writes a set of CGI scripts to track visitors to his online resume, he open sources the scripts and names them personal homepage.
  • In 97 Developers Andi Gutman and Zeev Suraski, in conjunction with the Lerdorf rewrote the existing parser and released PHP 3.0 renaming it hypertext preprocessor. Which it completed its revolution into a really powerful server-side processing language.
  • PHP was free and open source, it attracted wide community of Developers.
  • Because PHP paired with the open source tools Linux, Apache, and mySQL, it was very important.
  • By using PHP you can find PHP on almost all hosting plans.
  • Large pool of existing developers to work with and PHP is relatively easy to learn with a lot of available resources.

      Content Management Systems

  • CMS: Content Management Systems
    • An application designed to control the creation management, publishing, and archiving of a site's content.
  • CMS Benefits
    • Updating content can be done through WYSIWYG editor, making it easy for non-technical people to update site content.
    • Most CMS allow you to create user groups with various permission levels.
    • This flexibility allows user to create content, pages, and regions within the site without involving a designer.
    • Most CMS have plugins and extensions at powerful features to sites.
  • Exploring CMS options
    • Some CMS are hosted online and only require an account to start building hosted sites
    • Other CMS can be installed on your own web server.
    • Features and page customization very greatly between CMS, be sure to research the features you need.
    • Popular CMS include Joomla, Drupal, Movable Type, ModX, Radiant, WordPress, and Expression Engine.

      Content Delivery Network

  • CDN: Content Delivery Network
    • A system for delivering content over a distributor network of servers.
  • CDNs are largely used to serve static resources such as javascript libraries, video, or other site dependencies.
  • Types of CDNs
    • Some CDNs host a single type of resource.
    • CDNs like YouTube serve not only as delivery networks, but content aggregators as well.
    • You can choose between numerous free or commercial CDNs for dedicated service.
    • Popular CDNs include Akamai, CloudFare, Amazon's CloudFront, EdgeCast, and Windows Azure.

      4. Assembly Technologies

      JavaScript libraries

  • JavaScript Library
    • Collection of JavaScript methods and functions that make it easier to perform targeted tasks.
    • JavaScript libraries extend the functionality of native JavaScript. They range from the extremely focused sets like Date.js to the broader task-oriented libraries like jQuery.
  • Pros Of Using Libraries
    • Using libraries makes the process of writing JavaScript more efficient
    • You can often extend your own coding capabilities through libraries.
    • The code is widely used and tested, with contributors improving the code on a consistent basis.
  • Cons Of Using Libraries
    • Libraries increase the size of your code, and often add additional service request.
    • You can become tied to specific Library syntax.
    • Without knowing native JavaScript you're not always making an informed Choice as to when a library is appropriate.

      Frameworks and boilerplates

  • Frameworks & Boilerplates
    • Collection of prebuilt to CSS, HTML, and JavaScript files designed to speed development of sites.
  • Boilerplate is a set of templates built around a specific starting point or goal. Meant to provide a starting point for building sites or applications.
  • Framework is a collection of assets designed to help build sites are applications. This could include CSS grids, JavaScript libraries or helpful scripts, and HTML templates.

    CSS Predecessors

  • CSS preprocessor
    • Scripting language that extends the functionality of CSS and must be compiled into native CSS code before publishing.
  • Preprocessor benefits
    • They add additional functionality that CSS lacks.
    • Their syntax makes right and CSS more efficient.
    • Many browser inconsistencies are dealt with automatically when the CSS is processed.
  • Preprocessor cons
    • In team environments, or if the production code is shared, everyone will need to know the preprocessor syntax.
    • Processed CSS can be bloated and inefficient.
    • Additional layer of complexity to workflows.
